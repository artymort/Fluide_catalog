# FLUIDE — заметки

- Делаем tablet-first электронный каталог парфюмерии.
- Создана стартовая страница: зацикленное видео, синее затемнение, белый логотип и кнопка перехода.
- Добавлена адаптация заставки для планшета, ПК и узкого вертикального экрана.
- Подключены локальные файлы Mazzard H в начертаниях Light, Regular, Medium, SemiBold и Bold.
- Все тексты интерфейса используют начертание Regular (400).
- Проверено наличие всех подключенных файлов; визуальная проверка ожидает доступного окна браузера.
- Исправлены слои заставки: видео теперь находится перед фоном, а затемнение и интерфейс — поверх видео.
- Управление встроенным браузером подключено, но для проверки требуется открыть активное окно Browser/Preview в Codex.
- Проверен MP4: H.264, 1264×720. SVG оказался непрозрачным; для него включено наложение `multiply`, чтобы видео было видно.
- Добавлено обновление версии CSS в ссылке, чтобы браузер не показывал старый закэшированный стиль.
- Кнопка закреплена точно по центру экрана; положение логотипа рассчитывается отдельно.
- Получен визуальный референс каталога: светлый минимализм, фиолетовые акценты, крупные tablet-first элементы, карточная сетка и пошаговый подбор.
- Из брендбука взяты цвета: #434865, #F4F0E6, #9FB6DF, #EAB6A8, #ECB0B5.
- Создан фиксированный главный экран каталога: логотип, заголовок и семь разделов; прокрутка на планшете отключена.
- Кнопка на заставке подключена к главному меню `catalog.html`.
- Проект опубликован в публичном репозитории `artymort/Fluide_catalog` и развёрнут через GitHub Pages.
- Добавлен PWA-режим: манифест, Service Worker, иконки и полноэкранный запуск с главного экрана планшета.
- Исправлена обрезка контента на планшетах: высота берётся из фактической видимой области, добавлены fallback для старых Android/iPad и safe-area отступы.
- Mazzard H добавлен в PWA-кэш, чтобы типографика совпадала на Android, iPad и компьютере.
- Цвета карточек 02 и 03 переставлены: «Все ароматы» — голубая, «Коллекции» — светлая.
- Из таблицы загружено 99 ароматов: 34 женских, 24 мужских и 41 унисекс; доступны категории, концентрации и ноты.
- Исходные Excel/CSV исключены из публичного GitHub-репозитория; для сайта позже будет подготовлен отдельный JSON.
- Создан первый экран пошагового подбора `selection.html`; кнопка «Подобрать аромат» подключена к нему, выбор пола сохраняется для следующих шагов.
- Исправлено обновление установленной PWA: HTML загружается сначала из сети, новая версия Service Worker автоматически активируется и перезагружает приложение.
- Переработан экран первого вопроса: карточки уменьшены, контент выровнен влево, выделение упрощено, кнопке добавлено пространство; заголовок главного меню также выровнен влево.
- Подбор расширен до четырёх шагов: пол, направление нот, насыщенность и концентрация; выбранные параметры передаются в результаты.
- Из Excel автоматически формируется `fragrances.json` с 99 ароматами, уровнями нот и рассчитанными ольфакторными направлениями.
- Создан каталог результатов с поиском, сортировкой и фильтрами слева; карточки ведут на отдельные страницы ароматов.
- Создана динамическая карточка аромата с номером, названием, исходным направлением, характеристиками и пирамидой нот.
- Следующий этап — добавить реальные изображения продукции и уточнить алгоритм ранжирования рекомендаций.
- Раздел «Все ароматы» отделён от результатов подбора через режим `mode=all`: он показывает полный каталог, имеет собственный заголовок, возврат в главное меню и сортировку без рекомендаций; переходы из подбора используют режим `mode=selection`, а карточка аромата возвращает пользователя в исходный раздел.
- Для полного каталога создана отдельная страница `all.html` со статическим возвратом в главное меню; зависимость ссылки «назад» от JavaScript и PWA-кэша устранена, а общая логика фильтров и карточек сохранена в `results.js`.
- Раздел «Все ароматы» преобразован в общий «Каталог»: после 99 ароматов добавлена остальная продукция из файла цен, появился фильтр по типу товара и универсальные страницы товаров; значение `3240`, фирменный пакет и повторяющиеся позиции не импортируются, «Коллекции» оставлены на следующий этап.
- Публикация GitHub Pages переведена с нестабильного Legacy Pages на отдельный GitHub Actions workflow для надёжного развёртывания статического каталога из ветки `main`.
- Состояние общего каталога сохраняется в URL: после просмотра карточки товара возврат восстанавливает поиск, сортировку и выбранные фильтры.
- Для 91 доступного аромата подготовлены оптимизированные WebP-изображения с прозрачным фоном в `images/fragrances`; исходные JPEG остаются вне Git, обработка воспроизводится скриптом `prepare_fragrance_images.py` через BiRefNet.
- Сетка каталога подготовлена под изображения продукции: на широком экране показываются четыре более вертикальные карточки в ряд с увеличенной зоной флакона; на средних экранах остаются три колонки, на планшетах — две.
- Подготовленные изображения автоматически привязываются к ароматам по внутреннему номеру и отображаются в карточках каталога и на страницах ароматов; для восьми позиций без изображения сохраняется номер-заглушка.
- Для каталога используются отдельные облегчённые миниатюры 480×360 без тяжёлых CSS-фильтров; изображения жёстко ограничены визуальной областью карточки, полноразмерные WebP загружаются только на странице аромата.
- Большая карточка аромата получила единый светлый фирменный фон: кремовая основа с холодно-голубым ореолом вместо тёмного градиента, чтобы прозрачные флаконы сохраняли контраст и сочетались с интерфейсом сайта.
- На десктопе размер флакона в большой карточке ограничен 74% визуальной области с максимальными размерами 520×560 px; планшетное отображение оставлено без изменений.
- В каталоге убраны разноцветные градиенты карточек: все зоны с флаконами используют единый светлый полупрозрачный фон, совпадающий с фоном панели фильтров.
- Информационная часть карточек отделена от зоны изображения тонкой линией и более плотным светлым фоном, чтобы название, характеристики и теги визуально не сливались с флаконом.
- Стартовая видеозаставка убрана: корневая страница и установленная PWA сразу открывают `catalog.html`, а возврат из главного меню на заставку удалён.
- В общий PWA-скрипт добавлен Screen Wake Lock с автоматическим восстановлением после возвращения приложения на экран, чтобы планшет не выключал дисплей во время работы с каталогом.
- Для совместимости с уже установленной PWA главное меню перенесено непосредственно в `index.html`; `catalog.html` оставлен только как перенаправление, поэтому сохранённая старая точка запуска больше не может открыть заставку после обновления кэша.
- Главное меню перестроено в формат витрины: две крупные верхние карточки «Подобрать аромат» и «Все ароматы», четыре компактные нижние карточки, приветственный заголовок и статичный индикатор выбранных товаров с нулевым счётчиком справа от логотипа.
- Обе строки карточек главного меню выровнены по высоте, заголовок «Откройте свой аромат» центрирован, а персиковый цвет перенесён с «Хитов сезона» на «Парфюмерный гардероб».
- Проверка обновлений PWA больше не зависит только от первой загрузки: Service Worker проверяется при возвращении приложения на экран, при `pageshow` и каждые пять минут, поэтому для следующих версий переустановка приложения не требуется.
- В нижних карточках главного меню зарезервированы одинаковые зоны под названия и описания, поэтому все заголовки начинаются на одной высоте независимо от переноса на две строки.
- Для чтения с планшета увеличены подписи всех карточек главного меню и минимальный размер заголовков нижнего ряда; выравнивание текстовых зон сохранено.
- Подбор аромата перестроен в четыре шага: пол, случай, ноты и сезон; на каждом шаге добавлена кнопка «Пропустить», мультивыбор для случая/нот/сезона, индикатор текущего шага и счётчик подходящих ароматов. Пол в подборе расширяется так: мужские включают унисекс, женские включают унисекс, унисекс показывает только унисекс; новые поля `occasion` и `season` уже передаются в результаты, но не сужают выдачу до заполнения этих данных в базе.
- В `build_fragrances.py` добавлена автоматическая разметка ароматов по сценариям `everyday`, `evening`, `date`, `gym`, `walk` и сезонам `summer`, `autumn`, `winter`, `spring`; `fragrances.json` пересобран с этими массивами для всех 99 ароматов. Разметка основана на нотах, концентрации, проценте масла и точечных переопределениях для очевидных свежих/спортивных и тёплых вечерних ароматов; при появлении ручных данных из таблицы её можно заменить или уточнить.
- Счётчик в подборе теперь учитывает только ответы до текущего шага включительно, поэтому старые ответы из следующих шагов в `sessionStorage` больше не искажают количество на первом экране; строка счётчика опущена ниже и увеличена для чтения с планшета.
- Общий каталог разделён на четыре быстрых раздела рядом с заголовком: «Парфюм» (99 ароматов и 9 твёрдых парфюмов), «Уходовая косметика» (4 товара), «Товары для дома» (20 товаров) и «Автопарфюм» (1 товар). Для каждого раздела левая панель показывает собственные фильтры; в «Парфюме» используются вопросы подбора по получателю, случаю, нотам и сезону, а старые фильтры категорий, концентрации и насыщенности удалены. Поиск сохранён, выпадающая сортировка убрана, состояние раздела и фильтров записывается в URL.
- Логотип в главном меню, каталоге, результатах подбора, пошаговом подборе и карточке товара ведёт на главную страницу. В правой части шапок добавлена единая корзина с тем же статичным счётчиком `0`, что и на главном экране; ссылка «Главное меню» в каталожных шапках заменена на «Подобрать ароматы» с переходом на первый шаг подбора.
